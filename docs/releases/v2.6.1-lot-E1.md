# v2.6.1 — Lot E1 : Sécurité & Compliance Enterprise

## Objectif
Fermer les 3 chantiers sécurité critiques restants pour débloquer la commercialisation Enterprise.

## Livré

### Backend
- **`audit_logs_immutable`** : table append-only avec chaîne hash SHA-256 calculée par trigger `BEFORE INSERT`. UPDATE/DELETE refusés via RLS et privilèges révoqués.
- **`verify_audit_chain_integrity()`** : RPC retournant `{valid, broken_at, checked}` pour détecter toute manipulation.
- **`log_audit_event(...)`** : helper RPC pour journaliser depuis l'application.
- **`webhook_allowlist_domains`** : table CRUD super_admin avec colonnes `is_active` et `match_suffix`.
- **`is_url_allowed(text)`** : refuse IPs privées (10/8, 172.16/12, 192.168/16, 127/8, 169.254/16), loopback, link-local IPv6, et valide contre allowlist active (exact ou suffix).
- **`email_security_flags`** : table de quarantaine pour emails suspectés de phishing.
- **`review_email_security_flag(_flag_id, _decision, _notes)`** : RPC pour approuver/rejeter un flag.

### Edge Functions
- **`_shared/email-security.ts`** : module mutualisé de sanitization HTML (allowlist tags/attributs) et détection phishing (homoglyphes, mismatch anchor/href, shorteners non-allowlistés).
- **`trigger-email/index.ts`** : pipeline sanitize → flag-if-suspect → block-or-send. Les emails high/critical sont bloqués et journalisés.

### Frontend
- **`/admin/audit`** (`AdminAudit.tsx`) : page paginée du journal immuable, filtres acteur/action/entité, badge "Chaîne intègre" temps réel, export CSV.
- **`AdminEmails > onglet Security`** (`EmailSecurityFlagsTab.tsx`) : liste flags pending/approved/rejected avec dialog de revue et actions Valider/Confirmer le blocage.
- **`AdminSettings > onglet Allowlist webhooks`** (`WebhookAllowlistTab.tsx`) : CRUD allowlist avec switch is_active et toggle match_suffix.
- **AdminSidebar** : nouvelle entrée "Audit immuable" sous Logs.
