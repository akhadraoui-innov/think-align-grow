# v2.6.1 — Lot E3 — Notifications in-app & Command Palette globale

## Backend
- Table `notifications` (user_id, organization_id, type, title, body, link, severity, read_at) + RLS user-scoped + lecture SaaS team.
- Publication realtime + `REPLICA IDENTITY FULL`.
- RPCs : `notify_user`, `mark_notifications_read(_ids[])`, `mark_all_notifications_read()`.
- Triggers branchés :
  - `welcome` après insert profile.
  - `org.suspended` / `org.reactivated` après update organizations.status.
  - `certificate.earned` après insert academy_certificates.

## Frontend
- Hook `useNotifications` avec canal realtime filtré `user_id=eq.{uid}` + toast auto sur INSERT.
- `NotificationsDropdown` intégré dans `PortalShell` et `AdminShell` (badge unread, mark-read par item, mark-all-read).
- `CommandPalette` étendue : groupes Portail, Administration (gated `isAdmin`), Exploration, Piliers, Cartes.
- Bind `Cmd+K` / `Ctrl+K` global dans `PortalShell` et `AdminShell`.

## Audit
- Insertion `notify_user` → notification visible et toast immédiat.

- RLS testée : un user ne voit pas les notifs d'un autre.
