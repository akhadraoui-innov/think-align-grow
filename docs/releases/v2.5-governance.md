# Release v2.5 — Gouvernance, Traçabilité & Gestion utilisateurs

**Date :** Avril 2026
**Domaines impactés :** Organisations · Membres · Équipes · Permissions · Audit · Sécurité

---

## 🎯 Objectifs

Combler les trois écarts identifiés à l'audit v2.4 :
1. **Audit trail vide** : 5 écrans admin lisaient `activity_logs`, aucune écriture n'avait lieu.
2. **Équipes inutilisables** : la table `team_members` n'avait aucun écran de gestion.
3. **Pas d'invitations** : ajout d'utilisateur réservé au super_admin via accès DB direct.

---

## ✨ Nouveautés

### 1. Traçabilité automatique (niveau base de données)

Toutes les actions sensibles sont désormais tracées **automatiquement par triggers PostgreSQL**, sans dépendre du code applicatif :

| Domaine | Actions tracées |
|---|---|
| Organisations | `org.created` · `org.updated` · `org.deleted` |
| Membres org | `org.member.added` · `org.member.role_changed` · `org.member.removed` |
| Rôles plateforme | `user.role.granted` · `user.role.revoked` |
| Équipes | `team.created/updated/deleted` · `team.member.added/removed` |
| Crédits | `credits.earned` · `credits.spent` · `credits.adjusted` |
| Permissions | `permission.granted` · `permission.revoked` |
| Profils | `user.status.active` · `user.status.suspended` |

**Bénéfices :** conformité SOC2/ISO, audit client possible, journal exhaustif sans intervention développeur.

### 2. Système d'invitations complet

- **Table** `organization_invitations` (token UUID unique, expiration 7j, état accepté/révoqué).
- **Edge function** `send-invitation` : crée l'invitation, envoie un email branded via l'infra Lovable.
- **Page publique** `/invitation/:token` :
  - Si non connecté : propose connexion ou inscription avec email pré-rempli.
  - Si connecté avec le bon email : un seul clic pour rejoindre l'organisation.
  - Si connecté avec un autre email : message clair de mismatch.
- **UI admin** : bouton "Inviter" actif dans la fiche organisation, section "Invitations en attente" avec actions Renvoyer / Révoquer.
- **Sécurité** : seuls les admins de l'org ou la SaaS team peuvent inviter ; fonction `accept_invitation` SECURITY DEFINER.

### 3. Sécurité owner-protection

- Trigger `protect_last_owner` : impossible de **retirer** ou **dégrader** le dernier propriétaire d'une organisation. Erreur claire renvoyée à l'utilisateur.
- Menu de rôles complet dans la fiche membre (owner, admin, member, lead, facilitator, manager, guest).

### 4. Synchronisation email

- `handle_new_user` met désormais à jour `profiles.email` à la création.
- Backfill : tous les profils existants ont reçu leur email depuis `auth.users`.
- Colonne **Email** affichée dans `/admin/users`.

### 5. Présence en temps réel

- Hook `useLastSeenTracker` dans `AppShell` et `PortalShell` : `profiles.last_seen_at` est mis à jour à chaque navigation (throttle 30 s).
- Indicateur **"En ligne"** (point vert) dans `/admin/users` et fiche organisation pour les utilisateurs vus < 5 min.

### 6. Équipes opérationnelles

- Modal "Membres" dans `OrgTeamsTab` : sélection multiple via checkboxes des membres de l'organisation.
- Sélection d'un **lead d'équipe** parmi les membres choisis.
- Bulk insert/delete dans `team_members`.

### 7. UX admin enrichie

**`/admin/users`** :
- Filtres : Rôle, Organisation, Statut.
- Recherche par nom ou email.
- Compteur dynamique « X / Y filtrés ».

**Fiche utilisateur** :
- Badge statut coloré (actif vert / suspendu rouge).
- Bouton **Suspendre / Réactiver** en un clic.

### 8. Hardening sécurité

- **Leaked Password Protection (HIBP)** activé sur l'authentification.
- Bucket **`avatars`** rendu privé (un user voit ses fichiers ; SaaS team accès complet).
- Bucket **`academy-assets`** rendu privé (lecture par utilisateurs authentifiés uniquement).
