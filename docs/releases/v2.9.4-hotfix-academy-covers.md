# v2.9.4 — Hotfix Academy covers

**Date** : 2026-04-29  
**Type** : Hotfix régression (priorité haute)

## Symptôme

Toutes les images de couverture des parcours Academy étaient cassées :
- `/admin/academy/paths` : aucune cover affichée (fallback gradient partout)
- `/portal/academie/paths` : idem
- `/portal` (dashboard formations) : idem

## Cause racine

La migration de hardening Lot 7 (`20260423103833_bbbce425…sql`) avait passé le bucket `academy-assets` en privé :

```sql
UPDATE storage.buckets SET public = false WHERE id = 'academy-assets';
```

Conséquence : les URLs publiques déjà persistées dans `academy_paths.cover_image_url` (forme `…/storage/v1/object/public/academy-assets/covers/<id>.jpg`) renvoyaient 400/403.

État avant fix :
- 9 parcours / 19 avaient une URL valide
- 18 fichiers présents dans le bucket (intacts)
- 0 image affichée côté UI

## Correctif

Migration `20260429_hotfix_academy_covers_public` :

1. `storage.buckets.public = true` pour `academy-assets`
2. Suppression de la policy `academy_assets_authenticated_read`
3. Création d'une policy `academy_assets_public_read` (SELECT pour `public`)
4. Policies INSERT/UPDATE/DELETE inchangées (SaaS team only)

## Vérifications

- ✅ Migration appliquée
- ✅ Linter : 1 nouveau warning attendu (`Public Bucket Allows Listing` — comportement voulu pour assets de présentation)
- ⏳ À valider en preview : `/portal/academie/paths` doit ré-afficher les 9 covers existantes

## Impact sécurité

Aucun impact négatif :
- Les covers sont des assets de présentation (visibles dès la page parcours)
- L'écriture reste strictement réservée à la SaaS team
- Aucune donnée sensible dans le bucket
