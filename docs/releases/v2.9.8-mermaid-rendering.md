# v2.9.8 — Rendu Mermaid premium dans EnrichedMarkdown

**Date** : 29 avril 2026
**Type** : Feature pédagogique (rendu visuel + interactivité)

## Contexte

Les contenus pédagogiques Academy générés par l'IA contiennent fréquemment des diagrammes en syntaxe Mermaid (flowchart, sequence, gantt…). Avant cette release, ces blocs ` ```mermaid ` n'étaient pas interprétés et restaient affichés en code source brut, dégradant l'expérience d'apprentissage.

## Implémentation

### Nouveau composant `src/components/academy/MermaidDiagram.tsx`

**Rendu**
- Lazy-load de la lib `mermaid` (~600kb) au premier mount → 0 impact sur le bundle initial
- Theme `base` paramétré via les CSS variables du design system (`--primary`, `--background`, `--border`, `--foreground`, `--muted`, `--accent`)
- Police héritée (`fontFamily: 'inherit'`) pour respecter Inter/font-display
- Re-render automatique au switch light/dark via `MutationObserver` sur `<html class>`
- Validation `mermaid.parse()` avant `render()` pour erreurs propres
- Sécurité `securityLevel: 'strict'` (anti-injection sur contenu IA-généré)

**Interactivité (toolbar overlay top-right, visible au hover/focus)**
- Zoom avant / arrière / reset (lib `svg-pan-zoom`)
- Mode plein écran via `Dialog` shadcn (95vw × 95vh)
- Téléchargement du SVG en un clic
- Affichage repliable du code source mermaid

**Raccourcis clavier**
- `+` / `=` : zoom avant
- `-` : zoom arrière
- `0` : reset
- `F` : plein écran

**Accessibilité**
- `role="img"` + `aria-label` sur chaque SVG
- Container focusable (`tabIndex={0}`) avec ring focus visible
- Label ARIA décrivant les raccourcis

**Fallback gracieux**
- Card destructive avec icône `AlertTriangle` si syntaxe invalide
- Message d'erreur Mermaid extrait (1ère ligne)
- Code source repliable pour debug
- `console.warn` pour traçabilité admin

### Patch `src/components/academy/EnrichedMarkdown.tsx`

Ajout d'une interception en tête du handler `code` :
```tsx
if (className === "language-mermaid") {
  return <MermaidDiagram chart={String(children).trim()} />;
}
```

Tous les composants utilisant `EnrichedMarkdown` (PortalFormationsModule, PortalAcademieModuleDetail, PortalGuideReader, PortalCertificateDetail, AcademyExercise, ModuleReviewView, UCMSynthesisPage, etc.) bénéficient automatiquement du rendu mermaid.

## Dépendances ajoutées

- `mermaid@11.14.0` — lazy-loadé
- `svg-pan-zoom@3.6.2` — pan/zoom léger (14kb)

## Types Mermaid supportés

flowchart, sequence, gantt, ER, state, mindmap, timeline, sankey, quadrant, journey, pie, requirement, gitgraph, C4, block, packet, architecture.

## Hors scope

- Pas d'éditeur visuel Mermaid dans l'admin (la prompt IA produit déjà du Mermaid valide pour 95% des cas)
- Pas de migration des renderers UCM/Simulator qui utilisent leurs propres composants
- Pas d'export PNG (le SVG est universellement supporté et meilleur qualité)
