# v2.9.6 — Couvertures Academy : qualité premium + regen permanente

**Date** : 2026-04-29
**Type** : Correctif qualité IA + amélioration UX

## Contexte

Les couvertures générées depuis v2.9.4/v2.9.5 étaient devenues basiques (icône centrée sur dégradé plat) alors que les anciennes étaient des illustrations éditoriales riches (isométriques, scènes 3D corporate). Origine : le prompt système avait été simplifié à l'extrême ("clean illustration with a few simple flat icons … very clean composition, no clutter") et le modèle prompt-writer était `gemini-2.5-flash-lite` (trop court, trop générique).

De plus, le bouton de regénération n'apparaissait que sur les cartes SANS cover → impossible de remplacer une image jugée moche.

## Changements

### Style premium restauré

Dans `supabase/functions/academy-generate/index.ts`, nouveau brief commun à `generateCover` et `generateAllCovers` :

> Premium e-learning course cover, editorial corporate illustration of Behance/Dribbble quality. Rich, detailed scene combining isometric 3D elements and modern flat-vector business iconography (charts, devices, abstract data flows, silhouettes, tools) directly related to the topic. Cinematic lighting with soft glows and depth, layered foreground/background composition, vibrant but harmonious palette of 2-4 colors with one strong topic-driven accent, subtle gradients, polished shadows. Wide 16:9 aspect, professional and aspirational mood. Absolutely NO text, NO letters, NO words, NO watermark, NO logo.

- Modèle prompt-writer passé de `gemini-2.5-flash-lite` → **`gemini-2.5-flash`** (prompts plus riches, 60-90 mots).
- Modèle image inchangé : `google/gemini-3.1-flash-image-preview`.
- Contrainte "no text" préservée (mémoire `style/academy-path-visuals-standard`).

### Bouton "Régénérer la couverture" toujours accessible

Dans `src/pages/admin/AdminAcademyPaths.tsx` :
- Le bouton image (icône `ImageIcon`) apparaît **au hover sur toutes les cartes**, plus seulement sur celles sans cover.
- Tooltip dynamique : "Régénérer la couverture" (cover existante) / "Générer la couverture" (cover absente).
- `generateSingleCover` détecte le mode regen vs first-gen et adapte le toast (`loading` persistant + `success` final).
- Gestion soft des fallbacks 429/402 (toast warning, pas d'erreur bloquante).
- Cache-busting de l'`<img src>` via `?v=<updated_at>` → la nouvelle image s'affiche immédiatement après regen sans `Ctrl+F5`.

## Fichiers modifiés

- `supabase/functions/academy-generate/index.ts` (`generateCover` + `generateAllCovers`)
- `src/pages/admin/AdminAcademyPaths.tsx` (bouton regen + img cache-busting + UX toasts)
- `.lovable/memory/features/academy/path-visuals-ai.md`

## Hors périmètre

- Pas de regen auto en masse des covers existantes (l'utilisateur décide cover par cover via le nouveau bouton).
- Pas de champ "brief image custom" (toujours backlog v2.9.5).
