# v2.9.7 — Audit de parité Admin ↔ Portal

**Date** : 29 avril 2026
**Type** : Audit qualité (documentation seule, aucun code modifié)
**Périmètre** : Vérifier que les développements des releases v2.9.x ont bien été répliqués sur les deux surfaces (Cabinet `/admin/*` et Portail apprenant `/portal/*`).

## Contexte

Suite au bug remonté en v2.9.6 (bouton "Régénérer couverture" présent sur Admin mais absent du Portal), nous avons audité l'ensemble des paires Admin↔Portal pour détecter d'autres oublis de duplication potentiels sur les développements récents.

## Méthode

Extraction Python automatisée sur 5 axes pour chaque paire de pages :

1. Appels Edge Functions : `supabase.functions.invoke('...')`
2. Requêtes Supabase : `.from('table_name')`
3. Toasts utilisateur : `toast.success/error/info/loading`
4. Navigations : `navigate('/route')`
5. CTA : `<Button onClick=...>`

Calcul des différences symétriques (présent uniquement Admin / présent uniquement Portal) pour chaque paire.

## Résultats

### Verdict global : ✅ Système sain

10 paires sur 16 ont **exactement la même longueur en lignes** et 0 divergence fonctionnelle. Les développements récents (v2.9.5 auto-trigger, v2.9.6 regen, ainsi que les features antérieures booklet pédagogique, tutor inline, KPI normalization, certificats, persistance interactions) ont été correctement répliqués des deux côtés.

### Divergences confirmées comme volontaires (non corrigées)

| Paire | Écart Admin only | Écart Portal only | Justification |
|---|---|---|---|
| AdminUCM ↔ PortalUCM | tables `ucm_quota_usage`, `ucm_projects`, `ucm_use_cases` | — | Gestion des quotas réservée admin (cf. mémoire `ucm/monetization-quotas-admin-v2`) |
| AdminPracticeStudio ↔ PortalPratique | — | tables `practice_organizations`, `practice_user_assignments` | Filtrage par assignation utilisateur côté portail (RLS) |
| AdminWorkshops ↔ PortalWorkshops | — | navigation `/auth`, page d'entrée volumineuse | Le portail contient l'expérience d'entrée, l'admin est une page de gestion |
| AdminDesignInnovation ↔ PortalChallenges | — | navigation `/auth` | Idem |

### Écart cosmétique mineur restant (non bloquant)

| Paire | Écart | Décision |
|---|---|---|
| AdminAcademyPaths ↔ PortalAcademiePaths | `missingCount` (compteur de parcours sans couverture) + bouton batch "Générer toutes les couvertures manquantes" | Conservé admin-only volontairement. Action de bulk maintenance légitime côté gestion. |

## Hotfix appliqué dans le cadre de l'audit

**Catalogue Formations (`/portal/formations`)** : ajout du cache-busting `?v=${timestamp}` sur `getCoverImage` dans `src/pages/portal/PortalFormations.tsx`. Sans cela, le navigateur servait l'ancienne version cachée de la couverture IA, créant une incohérence entre le catalogue (image obsolète) et le détail/admin (image fraîchement régénérée). Désormais aligné sur la formule v2.9.6.

## Conclusion

Aucune dérive fonctionnelle critique détectée. Les v2.9.5 et v2.9.6 sont correctement synchronisées sur les deux surfaces depuis l'application des correctifs du jour.

## Mesure préventive

Mise à jour de la mémoire technique `mem://technical/portal-architecture-duplication` avec une **checklist de parité obligatoire** à exécuter pour toute PR touchant `src/pages/admin/*` ou `src/pages/portal/*`.
