# Release v2.6 — Email Platform GROWTHINNOV

**Date :** Avril 2026
**Domaines impactés :** Communication transactionnelle · Marketing · Branding · Multi-fournisseurs · Multi-tenant

---

## 🎯 Objectifs

Doter la plateforme d'un **Email Studio** complet :
1. Système d'envoi multi-fournisseurs (Lovable / Resend / SendGrid / SMTP).
2. Templates Markdown co-brandés (logo client + GROWTHINNOV) versionnés.
3. Automations événementielles (style n8n) paramétrables global et par organisation.
4. IA Marketing SaaS pour rédiger les emails et concevoir les déclencheurs.
5. 4 emails métier de référence : bienvenue, suspension, relance connexion, invitation.

## ✨ Nouveautés

### 1. Email Studio admin (`/admin/emails`)

5 onglets opérationnels :

| Onglet | Contenu |
|---|---|
| **Templates** | Liste filtrée global/org · éditeur split markdown + preview live · versioning automatique · assistant IA |
| **Automations** | Mapping `event → template` · conditions JSON · délai · activation par scope |
| **Logs & Stats** | KPIs 30j (envoyés, ouverts, cliqués, échoués) · filtres org/template/event/provider · export CSV |
| **Branding & Domaine** | Par org : upload logo, sender domain custom, opt-in tracking RGPD, période d'inactivité |
| **Fournisseurs Email** | Config par provider · credentials chiffrés · test d'envoi · override custom par org |

### 2. Architecture du dispatch

Edge Function **`trigger-email`** = cœur de résolution :

```
event + org → automation (org > global)
            → template (org > global)
            → provider (org > global, gated par plan/override)
            → branding (co-brandé si plan/override le permet)
            → render markdown→HTML
            → send via adapter
            → trace dans email_automation_runs
```

Idempotence assurée par `(trigger_event, entity_id, template_code)`.

### 3. Co-branding plan-based

| Plan | Logo dans le header | Sender domain | Provider custom |
|---|---|---|---|
| Starter | GROWTHINNOV uniquement | `noreply@growthinnov.com` | Lovable (Resend mutualisé) |
| Pro | GROWTHINNOV + logo client (si chargé) | `noreply@growthinnov.com` | Lovable |
| Enterprise | GROWTHINNOV + logo client | Custom (DNS DKIM/SPF) | Au choix : Resend / SendGrid / SMTP |

**Override par organisation** : la SaaS team peut activer chacun des 3 flags (`custom_email_domain`, `custom_email_provider`, `email_co_branding`) pour n'importe quelle org via `organizations.email_features_override` (jsonb).

### 4. Templates métier livrés

| Code | Déclencheur | Description |
|---|---|---|
| `welcome` | `user.created` (trigger SQL sur `profiles INSERT`) | Bienvenue + 10 crédits offerts + CTA Découvrir |
| `account_suspended` | `user.status.suspended` (trigger SQL sur `profiles UPDATE`) | Notification sobre, contact support |
| `login_reminder` | `user.inactive` (cron quotidien 09h UTC) | Relance après N jours d'inactivité (configurable par org, défaut 30j) — anti-spam : max 1 par 30j |
| `invitation` | EF `send-invitation` v2.5 (réorientée) | Invitation org branded |

### 5. IA Marketing SaaS

Edge Function **`email-marketing-ai`** (Lovable AI · `google/gemini-3-flash-preview` · streaming SSE) avec 3 modes :
- **`compose`** — génère subject + corps markdown + variables suggérées depuis un brief.
- **`refine`** — améliore un template existant (ton, CTA, mobile, longueur).
- **`automation_design`** — propose `trigger_event`, `conditions`, `delay_minutes` en JSON structuré.

**Accès gardé** par la permission DB `email.compose` (cohérent avec l'architecture v2.5).

UI : panneau ✨ Assistant IA dans l'éditeur de templates.

### 6. Multi-fournisseurs avec credentials chiffrés

`supabase/functions/_shared/email-adapters/` :
- `lovable.ts` — enqueue dans `pgmq` (queue existante).
- `resend.ts` — POST `https://api.resend.com/emails`.
- `sendgrid.ts` — POST `https://api.sendgrid.com/v3/mail/send`.
- `smtp.ts` — Deno SMTP (host/port/user/pass).

Interface commune `send({ to, from, subject, html, text }) → { success, messageId, error }`.

Credentials stockés dans `email_provider_configs.credentials_encrypted` via **pgcrypto** (`pgp_sym_encrypt`) ; déchiffrés uniquement dans la EF par RPC `decrypt_email_credentials`. Encryption helper admin `encrypt_email_credentials_admin` réservé à la SaaS team.

### 7. Tracking opt-in RGPD

Activable **par organisation** via `organizations.email_tracking_enabled`. Quand activé :
- Pixel d'ouverture inséré dans le HTML rendu.
- Réécriture des liens pour suivi des clics.
- Stocké dans `email_automation_runs.opened_at` / `clicked_at`.

Désactivé par défaut → conformité RGPD respectée pour les comptes Starter/Pro qui ne consentent pas explicitement.

- 6 nouvelles tables : `email_providers`, `email_provider_configs`, `email_templates`, `email_template_versions`, `email_automations`, `email_automation_runs`.

- Domaine de permissions `email.*` ajouté + assignations de rôles.

- Trigger versioning `capture_email_template_version`.
- Trigger DB `trg_email_welcome` (INSERT profiles).
- Trigger DB `trg_email_suspended` (UPDATE profiles.status).
- Cron `email-login-reminders` (quotidien 09h UTC) → `cron_dispatch_login_reminders()`.
- Fonctions pgcrypto `encrypt_email_credentials` + variante admin + `decrypt_email_credentials`.
- Helper SQL `dispatch_email_event` (POST `trigger-email` via `pg_net`).
- Bucket `brand-assets` (public read, write SaaS team).
- Seed : 4 providers actifs, 4 templates globaux, 4 automations globales.
