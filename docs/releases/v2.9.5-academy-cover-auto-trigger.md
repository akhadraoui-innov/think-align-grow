# v2.9.5 — Auto-trigger covers Academy

**Date** : 2026-04-29  
**Type** : Amélioration UX + automatisation

## Contexte

Suite au hotfix v2.9.4 qui a restauré la lecture publique du bucket `academy-assets`, 10 parcours restaient sans couverture (jamais générée). Ce patch ajoute une logique d'auto-déclenchement et améliore l'UX de génération.

## Changements

### Auto-trigger à la création / validation
- **Création manuelle** d'un parcours : génération de cover déclenchée automatiquement après l'insertion
- **Édition** d'un parcours : génération uniquement si `cover_image_url` est vide (pas de remplacement intempestif)
- **Génération IA** (`generate-path`) : cover déclenchée juste après la création AI
- Lancement **fire-and-forget** avec toasts info / success / error → le formulaire se ferme immédiatement, l'utilisateur n'attend pas

### Bannière contextuelle "Couvertures manquantes"
- Affichée au-dessus de la barre de filtres uniquement si N parcours sans cover
- Bouton **"Générer maintenant"** qui réutilise le batch existant `generate-all-covers`
- Disparaît automatiquement quand toutes les couvertures sont générées

### UX du bouton batch
- Toast `loading` persistant pendant le batch (10 parcours = 30-60s)
- Remplacement par un toast `success` ou `error` à la fin (même `id`)

## Pipeline réutilisé (inchangé)

- Edge function : `supabase/functions/academy-generate/index.ts`
- Actions : `generate-cover` (single) et `generate-all-covers` (batch des `cover_image_url IS NULL`)
- Modèle : `google/gemini-3.1-flash-image-preview`
- Style mémorisé : "Professional e-learning course cover, gradient 2-3 couleurs, icônes flat, **NO text**, 16:9"
- Upload : `academy-assets/covers/<path_id>.png` (upsert, lecture publique depuis v2.9.4)

## Fichiers modifiés

- `src/pages/admin/AdminAcademyPaths.tsx`
  - `upsert.mutationFn` retourne `{ savedId, needsCover, wasEdit }`
  - `upsert.onSuccess` déclenche `generate-cover` si nécessaire
  - `generateAI.onSuccess` déclenche `generate-cover` après création IA
  - Ajout de la bannière contextuelle "X parcours sans couverture"
  - `generateAllCovers` utilise `toast.loading` + `toast.success({ id })`

## Hors périmètre

- Pas de modification de l'edge function (pipeline déjà robuste)
- Pas de cron de régénération automatique (consommation crédits non maîtrisée)
- Pas de prévisualisation pré-validation (workflow asynchrone assumé)
